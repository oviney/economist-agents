# GitHub Copilot — Instructions for Economist-Agents

**This file is intentionally thin.** Project conventions live in a single source of
truth; this file only points at it so the rules don't drift across assistants.

## Single source of truth

- **`CLAUDE.md`** — operating mode, lifecycle discipline, coding standards (type hints,
  docstrings, `orjson`, `logger` not `print`, mock APIs in tests, >80% coverage), quality
  gates, and the backlog model. Read it first.
- **`skills/*/SKILL.md`** — the authoritative workflows:
  - `skills/economist-writing/SKILL.md` — the Economist voice (British spelling, no
    throat-clearing, data-first / cite every claim)
  - `skills/python-quality/SKILL.md` — full code standards
  - `skills/architecture-patterns/SKILL.md` — pipeline architecture (Anthropic Agent SDK)

## Architecture in one line

A 3-stage content pipeline (discovery → editorial board → generation) built on the
Anthropic Agent SDK (`src/agent_sdk/`), with deterministic quality gates. See
`docs/FLOW_ARCHITECTURE.md`.

> **Note:** The richer, machine-maintained Copilot context lives in
> `.github/copilot-instructions.md`, regenerated by `scripts/sync_copilot_context.py`
> (part of the closed-loop quality validator). Do not hand-edit that file.
